一些Spring面试题

你跺你也麻
2022-11-07
八股文

IoC

1. 什么是IoC

控制反转（Inversion of Control），将创建对象的控制权由程序员交给程序。好处在于我们不需要再关注对象的创建和相互的依赖关系，可以更加专注于业务逻辑。

2. 什么是DI

依赖注入（Dependence Injection），是控制反转的一种常见的实现方式。

3. 什么是Spring Bean

Spring Bean就是指被IoC容器管理的对象。我们需要告诉IoC容器要管理哪些对象，可以通过XML配置文件，注解或者Java配置类来说明。

4. @Component、@Configuration和@Bean的区别

@Component和@Bean都是用于声明Bean的注解，而@Configuration是用于声明配置类的注解，相当于xml中的<beans>。

  • @Component用于声明一个类是Spring Bean。
  • @Configuration注解标识的类中声明了一个或者多个@Bean方法，Spring容器可以使用这些方法来注入Bean。而@Configuration本身也被标注了@Component注解，因此被标识的类也会被注入为Spring Bean。

5. @Autowired和@Resource注解的区别

  • @Autowired 是 Spring 提供的注解，@Resource 是 JDK 提供的注解。

  • Autowired 默认的注入方式为byType（根据类型进行匹配），@Resource默认注入方式为 byName（根据名称进行匹配）。

  • 当一个接口存在多个实现类的情况下，@Autowired 和@Resource都需要通过名称才能正确匹配到对应的 Bean。Autowired 可以通过 @Qualifier 注解来显式指定名称，@Resource可以通过 name 属性来显式指定名称。

6. Bean的作用域有哪些，如何声明Bean的作用域

  • 单例（singleton），默认的作用域，也是最常用的，IoC容器中只存在一个对象实例。
  • 原型（prototype），每次注入bean都会创建一个新的bean实例。
  • 请求（request），仅web应用可用，每一次HTTP请求都会创建一个新的bean实例。
  • 会话（session），仅web应用可用，每一次来自新session的请求都会创建一个新的bean实例。
  • 全局（application/global-session），仅web应用可用，每一次来自新session的请求都会创建一个新的bean实例。
  • websocket，仅web应用可用，每一次WebSocket会话创建一个新的bean实例。

单例是默认的作用域。如果选择其他的作用域，要使用 @Scope 注解，他可以与 @Component 或者 @Bean 一起使用。

7. 单例 Bean 的线程安全问题

大部分 Bean 实际都是无状态（没有实例变量）的（比如 Dao、Service），这种情况下， Bean 是线程安全的。如果一定要给Bean增加状态，就在类中定义一个 ThreadLocal 成员变量，将需要的可变成员变量保存在 ThreadLocal 中。

8. ApplicationContext有哪些

子接口包括ConfigurableApplicationContext、WebApplicationContext、ConfigurableWebApplicationContext，实现类比较常见的包括ClassPathXmlApplicationContext、AnnotationConfigWebApplicationContext等等。

9. Bean的生命周期

  1. 读取配置文件，加载BeanDefinition
  2. BeanFactoryPostProcessor
  3. 实例化bean
  4. 填充属性
  5. 初始化bean
    1. 调用Aware方法（BeanNameAware、BeanClassLoaderAware、BeanFactoryAware、EnvironmentAware、EmbeddedValueResolverAware、ResourceLoaderAware、ApplicationEventPublisherAware、MessageSourceAware、ApplicationContextAware、ServletContextAware）
    2. 调用BeanPostProcessor的前置方法
    3. 调用bean定义信息中的init-method
    4. 调用BeanPostProcessor的后置方法
    5. 完成创建bean
  6. 销毁时，调用DestructionAwareBeanPostProcessors的postProcessBeforeDestruction方法
  7. 销毁时，调用bean定义信息中的destroy-method

10. 循环依赖问题是怎么产生的，如何解决，什么是三级缓存

  • 产生

    假设有A，B两个Bean，互相依赖。那么在A完成实例化之后，初始化A的过程中，需要B的实例，就会去IoC容器中查找B的实例，而此时B还没有被创建出来，就需要去创建B；而在B完成实例化之后，初始化B的时候，又需要A的实例，同样去IoC容器中查找A的实例，但此时A正在等待B完成创建，这样两个Bean都无法完成初始化，产生循环依赖问题。

  • 解决

    Spring是通过提前暴露对象的方式来解决循环依赖问题的，在上面的情况中，A完成实例化之后，在初始化之前，就提前暴露到容器中，这样在B实例化完成后，初始化的过程中，可以在容器中获取到A的不完整的实例，使得B的初始化能正常完成，从而使得A的初始化也能正常完成。

  • 三级缓存

    不考虑AOP的情况，只需要二级缓存即可。三级缓存是为了解决Bean使用代理之后的循环依赖问题。

11. BeanPostProcessor接口的作用和实现

  • 接口作用：扩展点，提供了让我们能在Bean初始化的之前（postProcessBeforeInitialization）和之后(postProcessAfterInitialization)修改bean的能力。
  • 实现举例：
    1. @Autowired：MergedBeanDefinitionPostProcessor->AutowiredAnnotationBeanPostProcessor
    2. @PostConstruct和@PreDestroy：CommonAnnotationBeanPostProcessor

12. BeanFactoryPostProcessor接口的作用和实现

  • 接口作用：扩展点，增强BeanDefinition
  • 实现举例：
    1. @Component：BeanDefinitionRegistryPostProcessor->ConfigurationClassPostProcessor（也就是自动装配的实现位置）

AOP

1. 什么是AOP

面向切面编程（Aspect-Oriented Programming），能够将那些与业务无关，却为业务模块所共同调用的逻辑或责任（例如事务处理、日志管理、权限控制等）封装起来，便于减少系统的重复代码，降低模块间的耦合度，并有利于未来的可拓展性和可维护性。

2. Spring的AOP是如何实现的

Spring AOP 就是基于动态代理的，如果要代理的对象，实现了某个接口，那么 Spring AOP 会使用 JDK Proxy，去创建代理对象，而对于没有实现接口的对象，就无法使用 JDK Proxy 去进行代理了，这时候 Spring AOP 会使用 Cglib 生成一个被代理对象的子类来作为代理。

Spring 中使用了哪些设计模式

  • 单例模式：bean默认都是单例的
  • 原型模式：bean的作用域可以指定为prototype
  • 代理模式：动态代理
  • 工厂模式：BeanFactory
  • 模板方法：postProcessBeanFactory、onRefresh、initPropertyValue
  • 策略模式：XmlBeanDefinitionReader、PropertiesBeanDefinitionReader
  • 观察者模式：listener、event、multicast
  • 适配器模式：Adapter
  • 包装器模式：BeanWrapper
  • 责任链模式：使用aop的时候会先生成一个拦截器链
  • 委托者模式：delegate，比如PostProcessorRegistrationDelegate
